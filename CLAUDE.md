# CLAUDE.md

Claude Code がこのプロジェクトで作業する際の指示。

---

## 役割

Claude Code は以下を担当する。**実装は行わない。**

- **Planning** — タスクの全体設計・分解
- **Repository understanding** — コード・ノート構造の把握と説明
- **Review** — 生成されたノートやスクリプトの品質確認
- **Task breakdown** — Codex への作業指示の作成
- **Note generation** — スライド・Colabをもとにした学習ノートの生成

---

## ノート生成の基本方針

### 言語ルール

- **出力言語**: 日本語メイン
- **英語を必ず併記するもの**: 専門用語、関数名、クラス名、ライブラリ名、モデル名
  - 例: 畳み込み層 `Conv2d`、過学習 `overfitting`、勾配降下法 `gradient descent`
- **英語のみでよいもの**: コードブロック内のコード、引数名、変数名
- **曖昧・不明な内容**: 推測せず `> ※ スライドに記載なし。要確認。` と明示する

### 入力の優先順位

1. **スライド（最優先）** — 講義の本質・概念説明の軸
2. **Colabノートブック** — 実装の詳細・コード例の補足
3. **Pythonファイル** — スクリプトの補足

### 構造ルール

- **大分類 → 中分類 → 詳細** の順に展開する
- 見出しは `#`〜`###` の3階層まで（深すぎない）
- 比較・対比があれば**必ず表にする**
- 手順・ステップがあれば**番号付きリスト**にする
- コードは必ず**コードブロック + 説明コメント**をセットにする
- 定義・重要な概念は `> 引用ブロック` で強調する

---

## ノートの必須セクション

毎回のノートに以下のセクションをすべて含める。詳細は `prompts/SKILL.md` を参照。

1. **概要ブロック**（Lecture番号・テーマ・対応Lab）
2. **本文**（スライドの流れに沿った内容）
3. **重要用語まとめ表**
4. **復習ポイント**
5. **混同しやすいポイント**
6. **理解チェック問題**

---

## 作業ルール

- `data/raw/` は読み取り専用。変更・削除しない
- ノートは `notes/obsidian/` に `LectureX_トピック名.md` 形式で保存する
- プロンプトテンプレートは `prompts/` に配置する
- ノート生成時は必ず `prompts/SKILL.md` のテンプレートに従う
- Codex への指示は具体的なファイルパスと期待する出力形式を含める

---

## 禁止事項

- `data/raw/` 配下のファイルの削除・上書き
- 危険な削除系コマンドの実行
- 推測による内容補完（不明な点は必ず明示する）
- テンプレートを無視した自由形式のノート生成
