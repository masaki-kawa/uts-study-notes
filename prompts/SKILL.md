# SKILL: 学習ノート生成 `generate-study-note`

このファイルは、UTSの授業資料から学習ノートを生成するための標準スキル定義。
Claude Code はノート生成時に必ずこのファイルを参照すること。

---

## トリガー

以下のいずれかの指示があったとき、このスキルを適用する：

- 「ノートを作って」「ノートを生成して」
- 「スライドからまとめて」「Colabをまとめて」
- 「LectureX のノートを作って」

---

## 入力の確認手順

ノート生成を開始する前に、以下を確認する：

1. `data/raw/slides/` にスライドPDFがあるか
2. `data/raw/colab/` に対応するノートブックがあるか
3. `data/raw/python/` にPythonファイルがあるか
4. Lecture番号・Lab番号・トピック名を特定する

入力がない場合はユーザーに確認する。

---

## 出力ファイル命名規則

```
notes/obsidian/LectureX_Lab Y_トピック名.md
```

例:
- `notes/obsidian/Lecture3_Lab3_CNN_Image_Classification.md`
- `notes/obsidian/Lecture5_Lab5_RNN_Sequence_Models.md`

## 各セクションの生成ルール

### 本文セクション（スライド準拠）

- **スライドの章立てに沿って**セクションを作る（スライドの流れ = ノートの流れ）
- 各スライドの「定義・説明・図・例」を日本語でまとめる
- スライドに図・表がある場合 → Markdown の表か箇条書きで再現する

### 表を使うべき場面

| 場面 | 例 |
|------|----|
| 複数の概念・手法を比較するとき | 多クラス vs 2値分類 |
| パラメータ一覧をまとめるとき | `nn.Conv2d` の引数 |
| 入出力・フォーマットを示すとき | 画像のサイズ変化 |
| 応用例・ユースケースを列挙するとき | Computer Vision の応用分野 |
| 用語・定義をまとめるとき | 重要用語まとめ表 |

### 比較表を作るべき場面

| 場面 | 例 |
|------|----|
| 2つ以上の手法・概念が登場したとき | CNN vs 通常NN |
| 「違い」「使い分け」の説明があるとき | グレースケール vs カラー正規化 |
| LectureとLabで同じ概念が異なる文脈で出てくるとき | スライドの説明 vs コード実装 |

### LectureとLabの橋渡し方法

スライドで説明された概念が、Labのどのコードに対応するかを明示する。

```markdown
> **Lecture → Lab**: スライドの「畳み込み演算」= Colabの `nn.Conv2d(...)` に対応。
```

コードブロックには必ず「何をしているか」「どの概念に対応するか」のコメントを添える：

```python
# [概念: 畳み込み層] フィルタサイズ3×3、パディング1でサイズを保持
self.conv1 = nn.Conv2d(in_channels=1, out_channels=32, kernel_size=3, padding=1)
```

### 専門用語の扱い方

- **初出時**: 日本語（英語）の形式で記載 → 畳み込み層（`Conv2d`）
- **2回目以降**: どちらか一方でよい
- **コードブロック内**: 英語のみ（コードはそのまま）
- **重要用語まとめ表**: 毎回必ず全用語を網羅する

### 日本語・英語の併記ルール

| 種別 | 書き方 | 例 |
|------|--------|----|
| 専門用語 | 日本語 `英語` | 過学習 `overfitting` |
| 関数・クラス名 | バッククォートのみ | `nn.Conv2d` |
| ライブラリ名 | バッククォートのみ | `PyTorch`, `torchvision` |
| モデル名 | そのまま英語 | VGG16, ResNet |
| 一般的な英語略語 | 日本語展開 + 略語 | CNN（畳み込みニューラルネットワーク） |

### 曖昧・不明な内容の扱い

- スライドに記載がない内容を推測で書かない
- 不明な点は以下の形式で明示する：

```markdown
> ※ スライドに記載なし。要確認。
```

- Colabにはあるがスライドにない内容は「補足」として区別する：

```markdown
> **補足（Lab）**: ここからはColabノートブックの補足内容。
```

## 出力品質チェックリスト

ノート生成後、以下をすべて確認してから保存する：

### 構造
- [ ] ファイル名が命名規則に従っている（`LectureX_LabY_トピック名.md`）
- [ ] 目次がすべてのセクションにリンクしている（Obsidianの `[[#...]]` 形式）
- [ ] 大分類 → 中分類 → 詳細の流れになっている
- [ ] 必須セクションが6つすべて含まれている

### 内容
- [ ] スライドの章立てに沿っている（スライド順 = ノート順）
- [ ] 定義・重要概念が引用ブロック（`>`）で強調されている
- [ ] LectureとLabの橋渡しコメントがある
- [ ] コードブロックに「概念」コメントが付いている

### 言語
- [ ] 専門用語に英語が併記されている
- [ ] 推測による補完をしていない（不明点は明示されている）
- [ ] 重要用語まとめ表がある

### 復習セクション
- [ ] 復習ポイントのチェックボックスがある
- [ ] 混同しやすいポイントの比較表がある
- [ ] 理解チェック問題が3問以上ある（解答例つき）

### Obsidian
- [ ] タグが末尾にある（`#科目 #Lecture-X #Lab-Y`）
- [ ] `[[#セクション名]]` 形式の内部リンクが機能する
