# AGENTS.md

Codex を含む実行系エージェント向けの作業ルール。

## 役割

実行系エージェント（主に Codex）は以下を担当する。

- **Implementation** — スクリプト・ノート生成コードの実装
- **Bug fixing** — スクリプトのエラー修正
- **Testing** — スクリプトの動作確認
- **Focused coding tasks** — Claude Code から渡された具体的な実装タスク

## 入出力パス

| 種別 | パス |
|------|------|
| スライド (入力) | `data/raw/slides/` |
| Colab (入力) | `data/raw/colab/` |
| Python (入力) | `data/raw/python/` |
| 処理済み中間データ | `data/processed/` |
| 学習ノート (出力) | `notes/obsidian/` |
| スクリプト | `scripts/` |

## ノート出力フォーマット

- フォーマット: Obsidian 対応 Markdown
- 言語: 日本語メイン、専門用語・関数名は英語を併記
- 構造: 大分類 → 中分類 → 詳細（見出し `#` `##` `###` で階層化）
- 要素: 箇条書き・表・コードブロックを積極的に使う

## 作業ルール

- `data/raw/` は**読み取り専用**。書き込み・削除禁止
- 新規ファイルは必ず適切なパスに配置する（ルート直下に置かない）
- スクリプトは冪等に設計する（何度実行しても同じ結果になるよう）
- 不明な仕様は推測せず、コメントで `TODO:` を記載して留める

## 禁止事項

- `data/raw/` 配下の変更・削除
- 危険な削除系コマンドの実行
- 未指示の機能追加
